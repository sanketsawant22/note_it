# 🌱 Spring Framework Basics (Java Web App)

## 🧩 Traditional Java Web Flow

```
Client Request → Servlet Server → Database
```

Inside a typical layered architecture:

```
Servlet (Controller) → Service → Repository (DAO)
```

### Example Structure:

```java
class Controller {
    Service service = new Service();
}

class Service {
    Repository repo = new Repository();
}
```

---

## ❌ Problem with Traditional Approach

* We manually create objects using `new`
* Tight coupling between classes
* Hard to manage dependencies
* Object lifecycle (creation + destruction) is unmanaged
* Too many objects → memory issues → heavy application
* Code becomes messy and hard to scale

---

## ✅ Solution: Spring Framework

👉 Spring says:

> "You focus on business logic, I’ll handle object creation and management."

---

## 🔁 IoC (Inversion of Control)

### 🔹 Definition:

Control of object creation and lifecycle is transferred from developer → Spring container.

### 🔹 Before Spring:

```java
Service service = new Service();
```

### 🔹 After Spring:

```java
@Autowired
Service service;
```

👉 Spring creates and injects the object for you.

---

## 💉 Dependency Injection (DI)

### 🔹 Definition:

Providing required dependencies (objects) to a class instead of creating them manually.

👉 Spring injects dependencies automatically.

---

## 🔗 Relationship Between IoC & DI

* **IoC = Principle**
* **DI = Implementation of IoC**

# 🧩 Spring Framework (XML Configuration)

## 📌 Core Idea

👉 In Spring Framework, we **tell Spring which objects (beans) to create** using a configuration file.

---

## ⚙️ Steps to Use Spring

### 1️⃣ Add Spring Dependencies

* Add Spring libraries (Maven/Gradle)

---

### 2️⃣ Create Configuration File

* Inside `resources` folder → create `spring.xml`

👉 This file contains rules for:

* Object creation
* Dependency Injection (DI)

---

### 3️⃣ Create IoC Container

```java
ApplicationContext context =
    new ClassPathXmlApplicationContext("spring.xml");
```

👉 This line:

* Creates **IoC Container**
* Loads all beans into memory

---

### 4️⃣ Get Bean from Container

```java
User obj = context.getBean("user", User.class);
obj.greet();
```

👉 No need to use `new` keyword

---

## 🧠 XML Bean Configuration

```xml
<beans>

    <bean id="user" class="demo.class.User"/>

    <bean id="laptop" class="demo.class.Laptop"/>

    <bean id="desktop" class="demo.class.Desktop"/>

</beans>
```

👉 Important:

* Even if you don’t use a bean → it still gets created (by default)

---

# 💉 Dependency Injection (XML Based)

---

## 1️⃣ Setter Injection

### XML

```xml
<bean id="user" class="demo.class.User">
    <property name="laptop" ref="laptop"/>
</bean>

<bean id="laptop" class="demo.class.Laptop"/>
```

### Java

```java
class User {
    private Laptop laptop;

    public void setLaptop(Laptop laptop) {
        this.laptop = laptop;
    }
}
```

---

## 2️⃣ Constructor Injection

### XML

```xml
<bean id="user" class="demo.class.User">
    <constructor-arg ref="laptop"/>
</bean>

<bean id="laptop" class="demo.class.Laptop"/>
```

### Java

```java
class User {
    private Laptop laptop;

    public User(Laptop laptop) {
        this.laptop = laptop;
    }
}
```

---

# 🔄 Autowiring (XML)

```xml
<bean id="user" class="demo.class.User" autowire="byType"/>
```

### Types:

* `byType` → match using class type
* `byName` → match using bean name

---

## ⚠️ Problem: Multiple Beans (Ambiguity)

```java
class User {
    private Computer com;

    public User(Computer com) {
        this.com = com;
    }
}
```

```xml
<bean id="user" class="demo.class.User" autowire="byType"/>

<bean id="laptop" class="demo.class.Laptop"/>
<bean id="desktop" class="demo.class.Desktop"/>
```

👉 Both `Laptop` and `Desktop` implement `Computer`

❌ Spring gets confused → **which one to inject?**

---

## ✅ Solution (XML): primary

```xml
<bean id="laptop" class="demo.class.Laptop" primary="true"/>
```

👉 Now `Laptop` will be injected by default

---

# 🚀 Spring Boot (Annotation-Based Approach)

👉 Spring Boot removes XML and does everything using annotations

---

## ⚙️ No XML Needed

Instead of:

```xml
<bean id="user" class="demo.class.User"/>
```

We write:

```java
@Component
class User {}
```

---

## 🧠 IoC + DI in Spring Boot

```java
@Component
class User {

    private final Laptop laptop;

    @Autowired
    public User(Laptop laptop) {
        this.laptop = laptop;
    }
}
```

```java
@Component
class Laptop {}
```

👉 Spring Boot:

* Creates objects automatically
* Injects dependencies

---

# 💉 Types of DI (Spring Boot)

---

## 1️⃣ Constructor Injection ✅ (Best)

```java
@Component
class User {

    private final Laptop laptop;

    @Autowired
    public User(Laptop laptop) {
        this.laptop = laptop;
    }
}
```

---

## 2️⃣ Setter Injection

```java
@Component
class User {

    private Laptop laptop;

    @Autowired
    public void setLaptop(Laptop laptop) {
        this.laptop = laptop;
    }
}
```

---

## 3️⃣ Field Injection ❌ (Not Recommended)

```java
@Component
class User {

    @Autowired
    private Laptop laptop;
}
```

---

# ⚠️ Problem: Multiple Beans (Spring Boot)

```java
@Component
class PaytmPaymentService implements PaymentService {}

@Component
class PhonePePaymentService implements PaymentService {}
```

```java
@Component
class OrderService {

    @Autowired
    private PaymentService paymentService; // ❌ ERROR
}
```

👉 Error:

```
NoUniqueBeanDefinitionException
```

---

# ✅ Solution 1: @Primary

```java
@Component
@Primary
class PaytmPaymentService implements PaymentService {}
```

👉 Default bean

---

# ✅ Solution 2: @Qualifier

```java
@Component("phonepe")
class PhonePePaymentService implements PaymentService {}
```

```java
@Component
class OrderService {

    @Autowired
    @Qualifier("phonepe")
    private PaymentService paymentService;
}
```

👉 Specific bean injection

---

# ⚡ Best Practice

```java
@Component
class OrderService {

    private final PaymentService paymentService;

    @Autowired
    public OrderService(@Qualifier("phonepe") PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}
```

---

# 🧠 Important Rules

* `@Primary` → default bean
* `@Qualifier` → specific bean
* If both present → **@Qualifier wins**
