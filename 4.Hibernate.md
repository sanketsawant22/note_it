

# 1. Why JDBC Was Created

Before JDBC, every database vendor had its own API.

For example:

* MySQL → MySQL API
* Oracle → Oracle API
* PostgreSQL → PostgreSQL API

Java applications would become database-dependent.

To solve this, Java introduced JDBC.

JDBC provides common interfaces:

```java
Connection
Statement
PreparedStatement
ResultSet
```

Every database vendor provides a driver implementing these interfaces.

So your code remains mostly unchanged.

```java
Connection con =
DriverManager.getConnection(url,user,password);
```

Whether it's MySQL or Oracle, JDBC API remains same.

# 5. Statement vs PreparedStatement vs CallableStatement

## Statement

```java
Statement st = con.createStatement();

st.executeQuery(
    "SELECT * FROM users WHERE id=10"
);
```

Problems:

* SQL injection
* Parsing every time

---

## PreparedStatement

```java
PreparedStatement ps =
con.prepareStatement(
    "SELECT * FROM users WHERE id=?"
);

ps.setInt(1,10);
```

Advantages:

### Security

User input never directly joins SQL.

### Better Performance

Query gets precompiled.

### Cleaner Code

Most used in projects.

---

## CallableStatement

Used for Stored Procedures.

```java
CallableStatement cs =
con.prepareCall("{call getUsers()}");
```

Used rarely compared to PreparedStatement.

# 6. SQL Injection

Very important.

Bad code:

```java
String username =
"' OR '1'='1";

String sql =
"SELECT * FROM users WHERE username='"
+ username + "'";
```

Generated SQL:

```sql
SELECT * FROM users
WHERE username='' OR '1'='1'
```

Returns all users.

Security issue.

PreparedStatement solves this.

```java
PreparedStatement ps =
con.prepareStatement(
"SELECT * FROM users WHERE username=?"
);
```

# 8. JDBC Transactions

Suppose:

```text
Account A -> Account B
```

Need:

1. Deduct money
2. Add money

Both must happen together.

---

Without transaction:

```java
update1();
System.out.println(10/0);
update2();
```

Money deducted.

Money not credited.

Database inconsistent.

---

Transaction:

```java
con.setAutoCommit(false);

try
{
   update1();
   update2();

   con.commit();
}
catch(Exception e)
{
   con.rollback();
}
```

---

Important Methods

```java
commit()
rollback()
setAutoCommit(false)
```

# 12. What is ORM?

ORM = Object Relational Mapping

Maps:

```text
Java Class <-> Table
Object     <-> Row
Field      <-> Column
```

Example:

### Table

```sql
users
```

| id | name   |
| -- | ------ |
| 1  | Sanket |

---

### Java

```java
User user
```

Hibernate automatically maps both.

---

JDBC Version

Fetch user with id = 1

Connection con =
DriverManager.getConnection(url,user,password);

PreparedStatement ps =
con.prepareStatement(
    "SELECT * FROM users WHERE id=?"
);

ps.setInt(1,1);

ResultSet rs = ps.executeQuery();

if(rs.next()){
    User user = new User();

    user.setId(rs.getInt("id"));
    user.setName(rs.getString("name"));
}

con.close();

Lots of boilerplate 😵‍💫

Hibernate Version
1. Entity Class
import jakarta.persistence.Entity;
import jakarta.persistence.Id;

@Entity
public class User {

    @Id
    private int id;

    private String name;

    private String email;

    public User() {}

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}
2. Fetch User
SessionFactory factory =
new Configuration()
        .configure()
        .addAnnotatedClass(User.class)
        .buildSessionFactory();

Session session =
factory.openSession();

User user =
session.get(User.class, 1);

System.out.println(user.getName());

session.close();
factory.close();

That's it 🔥

No SQL.

No PreparedStatement.

No ResultSet.

No manual mapping.

# 15. JPA vs Hibernate

Many beginners confuse these.

## JPA

Specification.

Set of rules.

Created by Java.

Examples:

```java
@Entity
@Id
@OneToMany
```

come from JPA.

---

## Hibernate

Implementation of JPA.

```text
JPA = Interface
Hibernate = Class implementing it
```

Similar to:

```java
List list = new ArrayList();
```

Here:

```text
List = JPA
ArrayList = Hibernate
```

Interview question:

**Is Hibernate same as JPA?**

No.

Hibernate implements JPA.

# 17. First-Level Cache

Hibernate automatically caches objects inside Session.

```java
User u1 =
session.get(User.class,1);

User u2 =
session.get(User.class,1);
```

Second call doesn't hit database.

Uses Session cache.

# 18. Lazy Loading vs Eager Loading

Suppose:

```java
User
 |
Orders
```

---

## Eager

```java
User + Orders
```

Loaded immediately.

---

## Lazy

```java
User
```

Orders loaded only when needed.

```java
user.getOrders();
```

Database call happens here.

Lazy is usually preferred.

# 23. Real Industry Flow (Most Common)

```text
Spring Boot
     |
Spring Data JPA
     |
Hibernate
     |
JDBC
     |
MySQL/PostgreSQL
```

In modern Spring Boot projects:

* You rarely write JDBC directly.
* You rarely use Session directly.
* Most work is done using Spring Data JPA repositories.

Example:

```java
@Repository
public interface UserRepository
       extends JpaRepository<User,Integer>
{
}
```

Then:

```java
userRepository.save(user);

userRepository.findById(1);

userRepository.deleteById(1);
```

No SQL.
No PreparedStatement.
No ResultSet.
