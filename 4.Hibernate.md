# 🔥 What JDBC is

## JDBC = Java Database Connectivity

It’s basically a **standard API (set of interfaces)** provided by Java to talk to relational databases.

* 👉 Java does **NOT** know MySQL, PostgreSQL, Oracle directly
* 👉 It only knows **interfaces**
* 👉 DB vendors provide **drivers (JAR files)** that implement those interfaces

---

# 🔌 Step-by-step JDBC Flow (core you MUST know)

## This is the heart of JDBC:

### 1. Load Driver (older way)

```java
Class.forName("com.mysql.cj.jdbc.Driver");
```

> 👉 Nowadays not required (auto-loaded)

---

### 2. Get Connection

```java
Connection con = DriverManager.getConnection(
    "jdbc:mysql://localhost:3306/dbname",
    "username",
    "password"
);
```

👉 This connects Java → Database

---

### 3. Create Statement

❌ Bad (avoid for dynamic queries)

```java
Statement stmt = con.createStatement();
```

✅ Best (use this)

```java
PreparedStatement ps = con.prepareStatement(
    "SELECT * FROM users WHERE id = ?"
);
ps.setInt(1, 10);
```

👉 Prevents SQL Injection + better performance

---

### 4. Execute Query

```java
ResultSet rs = ps.executeQuery();
```

---

### 5. Process Result

```java
while (rs.next()) {
    System.out.println(rs.getString("name"));
}
```

---

### 6. Close Resources

```java
rs.close();
ps.close();
con.close();
```

👉 Or use **try-with-resources (BEST PRACTICE)**

---

# ⚠️ Types of Statements (VERY IMPORTANT)

### 1. Statement

* Simple queries
* Not safe
* No parameters

### 2. PreparedStatement ✅ (MOST USED)

* Precompiled
* Secure
* Supports `?`

### 3. CallableStatement

* Used for stored procedures

# 🧪 Minimal Real Example

```java
import java.sql.*;

public class Demo {
    public static void main(String[] args) throws Exception {

        String url = "jdbc:mysql://localhost:3306/test";
        String user = "root";
        String pass = "1234";

        try (Connection con = DriverManager.getConnection(url, user, pass);
             PreparedStatement ps = con.prepareStatement("SELECT * FROM users");
             ResultSet rs = ps.executeQuery()) {

            while (rs.next()) {
                System.out.println(rs.getString("name"));
            }
        }
    }
}
```
---In case of JDBC, we have to write SQL queries manually and also insert (append) data into those queries ourselves.

For example, if we want to fetch a user:

```java
PreparedStatement ps = con.prepareStatement(
    "SELECT * FROM users WHERE id = ?"
);
ps.setInt(1, 10);
```

👉 Here:

* We are writing the SQL query manually
* We are also setting the values manually (`ps.setInt`)

This becomes repetitive and harder to manage as the project grows.

---

Because of this, we use **Hibernate (ORM – Object Relational Mapping)**.

👉 Hibernate removes the need to write most SQL queries manually.
👉 It allows us to work with Java objects instead of database tables.

For example:

```java
User user = session.get(User.class, 10);
```

👉 Here:

* No SQL is written
* Hibernate automatically converts this into a SQL query
* It also maps the result to a Java object

# 🔥 Hibernate Notes (Clean + Copy Ready)

---

# 📌 1. What is Hibernate?

**Hibernate = ORM (Object Relational Mapping)**

👉 Maps:

* Java Class ↔ Database Table
* Object ↔ Row
* Field ↔ Column

👉 Removes need to write most SQL manually

---

# 📌 2. Core Idea

```text
Java Object → Hibernate → JDBC → Database
```

👉 Hibernate sits on top of JDBC
👉 It internally generates SQL

---

# 📌 3. Basic Example

### Table: users

| id | name   |
| -- | ------ |
| 1  | Sanket |

---

### Java Entity

```java
import jakarta.persistence.*;

@Entity
@Table(name = "users")
public class User {

    @Id
    private int id;

    private String name;
}
```

# 📌 5. Hibernate Architecture

```text
Your Code → Session → Hibernate → JDBC → Database
```

---

## Main Components

### 1. SessionFactory

* Heavy object
* Created once
* Thread-safe

```java
// Create SessionFactory
        SessionFactory factory = new Configuration()
                .configure("hibernate.cfg.xml")
                .addAnnotatedClass(User.class)
                .buildSessionFactory();
```

---

### 2. Session

* Used to interact with DB
* Similar to JDBC Connection

```java
 // Create Session
        Session session = factory.openSession();
```

---

### 3. Transaction

* Required for DB changes

```java
Transaction tx = session.beginTransaction();
```

---

# 📌 6. CRUD Operations

---

## ✅ Insert

```java
Session session = factory.openSession();
Transaction tx = session.beginTransaction();

User user = new User();
user.setName("Sanket");

// session.save(user); in modern Hibernate (with JPA style), session.save() is not preferred
// session.persist(user);

tx.commit();  // actually saved in DB
session.close();  // connection closed
```

---

## ✅ Fetch

```java
User user = session.get(User.class, 1);
```

---

## ✅ Update

```java
User user = session.get(User.class, 1);
user.setName("Updated");

// ❌ Old
session.update(user);

// ✅ Modern
session.merge(user);
```

---

## ✅ Delete

```java
User user = session.get(User.class, 1);
// ❌ Old
session.delete(user);

// ✅ Modern
session.remove(user);
```

# 📌 7. Important Annotations

```java
@Entity                      // class = table
@Table(name = "users")       // table name

@Id                          // primary key
@GeneratedValue(strategy = GenerationType.IDENTITY)

@Column(name = "name")       // column mapping
```

---

# 📌 8. HQL (Hibernate Query Language)

👉 Works on objects, not tables

```java
Query q = session.createQuery("from User where name = :name");
q.setParameter("name", "Sanket");

List<User> list = q.list();
```

# 📌 10. Fetch Types

* Lazy → Load when needed
```java
  User user = session.byId(User.class).getRefrence(101);
```
* Eager → Load immediately
```java
  User user = session.get(User.class, 1);
```

# 📌 12. Hibernate vs JDBC

| Feature           | JDBC   | Hibernate |
| ----------------- | ------ | --------- |
| SQL               | Manual | Automatic |
| Boilerplate       | High   | Low       |
| Control           | High   | Medium    |
| Development Speed | Slow   | Fast      |

---

# 📌 13. Key Points to Remember

* Hibernate uses JDBC internally
* You work with objects, not SQL
* Session = main interface
* Transaction is required for DB changes
* Annotations replace XML config
